Why Cache SQL Queries?

Some SQL queries, as you've likely discovered, are expensive to execute. Complex queries of large databases, even when optimized, can take a significant amount of time -- so much so that it can inhibit pleasant UX. By caching complex queries which occur often, we can optimize-away the long wait times associated with those expensive SQL queries which occur the most often.

How Can I Cache SQL Queries?

SQL queries can be cached in one of two ways -- they can be cached SQL-server side, or 'locally' inside the PHP script which is querying the database. In this demo, we'll explore both ways to use caching assuming you've already built a PDO object called db which connects to your SQL database. It is assumed that a reader is running Ubuntu 14, has the LAMP stack installed, and has a reasonable level of aptitude with the Linux terminal.

Caching on the SQL server

The first method is to have your SQL database cache the query internally using SQL 'hints' inside of database queries. This can be achieved using the SQL_CACHE hint, which, when wrapped inside an SQL SELECT query, tells the DBMS to save the output of that query for future use.

//The SQL_CACHE flag at the start of the query tells SQL to cache the result.
$select = $db->query("SELECT SQL_CACHE * FROM table");

Subsequent queries to the database with the same query meaning (e.g. anything which provides the same functionality as 'select * from table' in this case) will return the cached result from the SQL server unless something has changed on the server side.

Caching in the PHP environment

The second method is to save the results of a SQL query inside an associative array in the cache of a PHP process. This can be achieved using a library called Memcached which allows a PHP user to store data in local cache.

Installing Memcached

Installing Memcached is very simple. Simply use any package manager to install php5-memcached. For example, use sudo apt-get install php5-memcached. This will automatically install and configure Memcached. A more in depth tutorial on Memcached and its settings can be found on Digital Ocean's help site, but for this example you won't need to modify any Memcached settings.

Using Memcached

In the following code snippet a Memcached object is constructed and connected 

$cache = new Memcached(); //Build a new memcached object
$cache->addServer('localhost', 11211); //11211 should always be the second parameter
    //(read the Memcached settings article above for a more in-depth explanation)

$key = 'some SQL query'; //This is the key for a key-value pair in the cache object
$result = $cache->get($key); //See if there's data for this query in the cache
